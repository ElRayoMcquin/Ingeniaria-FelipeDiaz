Caracteristicas JSON



JSON
  1. JSON es solo un formato de datos.
  2. Requiere usar comillas dobles para las cadenas y los nombres de propiedades.
  3. Las comillas simples no son válidas.
  4. Una coma o dos puntos mal ubicados pueden producir que un archivo JSON no funcione.
  5. Puede tomar la forma de cualquier tipo de datos que sea válido para ser incluido en un JSON, no solo arreglos u objetos. Así, por ejemplo, una cadena o un número único podrían ser objetos JSON válidos.
  6. A diferencia del código JavaScript, en el que las propiedades del objeto pueden no estar entre comillas, en JSON solo las cadenas entre comillas pueden ser utilizadas como propiedades.
